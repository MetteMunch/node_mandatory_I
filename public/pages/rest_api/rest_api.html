REST API

REST API, som står for Representational State Transfer Application Programming Interface, er et sæt arkitektoniske regler og principper, der bruges til at designe og udvikle webtjenester. Det er en type API, der tillader forskellige systemer og applikationer at kommunikere og interagere med hinanden over internettet.

Konceptet bag REST API er, at det behandler ressourcerne i en webapplikation, som om de var objekter i en datamodel, og det tillader klienter (såsom webbrowsere eller mobilapplikationer) at udføre operationer på disse ressourcer ved hjælp af standard HTTP-metoder som GET, POST, PUT, PATCH og DELETE. Hver ressource i en REST API er unikt identificeret af en URL, og klienter kan interagere med disse ressourcer ved at sende HTTP-anmodninger til de relevante URL'er.

Client sender en anmodning (request)

Request indeholder HTTP-metode, der specificerer handlingen. Eventuelt en header (fx med authentacation-token eller lignende) og en body med data (fx ved POST og PUT).

Server behandler og returnerer et svar (response)

Response indeholder en statuskode og body med det ønskede data (formatteret i JSON eller XML/HTML)
