CSR vs SSR

Parameter CSR - Client-side Rendering SSR - Server-side Rendering
Load time Langsommere end SSR ved første load af applikation, da browseren skal downloade, parse og eksekvere JS for at bygge DOM'en Hurtigere end CSR. Serveren genererer færdig HTML og sender til browser, så hurtig visning (initial load time).
Ressourcer Her er det klienten (browseren) som gør arbejdet med (rendering og og datahentning), så tungt specielt ved initial load. Her er det serveren som henter data og genererer en ny HTML ved hver request.
SEO Dårlig medmindre der laves en speciel opsætning. God til SEO, da alt indhold er klar ved load og nemt kan indekseres.
CORS API-kald sker i browseren direkte til ekstern API. Browser håndhæver CORS, så hvis ikke der er konfigureret korrekt kan fetch kald fejle. API kald sker server til server, så CORS er ikke et problem.
